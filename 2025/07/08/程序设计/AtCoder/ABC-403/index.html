# 题目大意

给定一个 A=(a1,...,an)A = (a_1, ..., a_n)A=(a1​,...,an​) 序列和一个非负整数 DDD，你现在要做的是，从 AAA 中删除尽可能少的元素，得到序列 BBB，需要满足：

  • 对于任意 i,j(1≤i<j≤∣B∣)i, j~(1 \le i < j \le |B|)i,j (1≤i<j≤∣B∣)，满足 ∣Bi−Bj∣≠D|B_i - B_j| \neq D∣Bi​−Bj​∣=D
    问：需要最少删除的元素个数是？

# 数据范围

  • 1≤N≤2×1051 \le N \le 2 \times 10^51≤N≤2×105
  • 0≤ai,D≤1060 \le a_i, D \le 10^60≤ai​,D≤106

# 题解

从考虑两个数 ∣Bi−Bj∣≠D|B_i - B_j| \neq D∣Bi​−Bj​∣=D 变成，将这个序列变为对于每个数，删到不存在比他大 DDD 和小 DDD 的数

我们此时，暂且只考虑对于 iii 来说，小 DDD 的，不考虑另一半
那么考虑使用 dpdpdp， dp[i][0 / 1] 表示不删除 / 删除值为 iii 的数后，要满足题意的最小操作次数

  • 先行预处理 i=minn→i=minn+d−1i=minn \to i = minn + d - 1i=minn→i=minn+d−1，这一段数没有 i−di - di−d 的前驱，故初始化
    • dp[i][1] = cnt[i] ，值为 iii 的删掉就是把所有 iii 都删掉
  • 再考虑 i=minn+d→i=maxxi = minn + d \to i = maxxi=minn+d→i=maxx
    • 如果不删除 iii
      • dp[i][0] = min(dp[i - d][0] + cnt[i - d], dp[i - d][1])
      • 取 （i−di-di−d 不删的次数 + i−di - di−d 的个数） 和 （i−di-di−d 删掉次数） 两者取 minnminnminn
    • 如果删除 iii
      • dp[i][1] = min(dp[i - d][0], dp[i - d][1]) + cnt[i]
      • 无论是否删除 i−di - di−d，iii 都是要删的，故需要加上 cnt[i] ，再加上 （i−di-di−d 删的次数）和（i−di-di−d 不删的次数）取 minnminnminn 即可

最后再从 maxxmaxxmaxx 倒着求到 maxx−d+1maxx - d + 1maxx−d+1，因为我们之前考虑的是， dp[i][0 / 1] 只考虑了，比 iii 小 ddd 的不会冲突，所以倒着找也不会出现大 ddd 冲突的情况 ``

# E

# 题目大意

你现在有两个空字符串集合 XXX 和 YYY，有 QQQ 个操作，每次操作会给你一个整数 TTT 和字符串 SSS，你现在需要做的是

  • 如果 TTT 为 111，则把 SSS 插入集合 XXX
  • 如果 TTT 为 222，则把 SSS 插入集合 YYY
    最后问：YYY 中有多少个字符串，不以 XXX 中的任意一个字符串为前缀

# 数据范围

  • 1≤Q≤2×1051 \le Q \le 2 \times 10^51≤Q≤2×105
  • ∑i=1Q∣Si∣≤105\sum\limits_{i=1}^{Q} |S_i| \le 10^5i=1∑Q​∣Si​∣≤105

# 题解

这题很明显考察的是字典树

  • clear[i] 记录 iii 位置是否被清除标记
  • val[i] 记录 iii 位置的值
  • res[i] 记录 iii 结点和其子结点的所有满足题意的字符串数量

现在写好了一个 updateupdateupdate 更新操作，其实现是这样的：

  1. 如果 clear[root] 为 111，代表已经被清除，故显然 res[root] = 0
  2. 反之，累加 res[0] -> res[26] 和 val[root]
  • 如果是 111 操作的话，那么标记 clear[结尾最后一个字符] 为 111
  • 如果是 222 操作的话，那么就需要把 val[结尾最后一个字符] 加一
