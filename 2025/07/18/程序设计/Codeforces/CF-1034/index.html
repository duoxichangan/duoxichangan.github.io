# A

# 题目大意

黑板上有 0→n−10 \to n - 10→n−1 的整数，在一轮游戏中，小 AAA 需要

  • 选取一个 aaa 并擦除
  • 然后小 BBB 选取一个 bbb，满足 a+b≡3(mod4)a+b \equiv 3 \pmod 4a+b≡3(mod4) 并擦除

这个游戏从小 AAA 开始，如果有一方不能操作了，则另一个人赢了，给你 nnn，请问谁能赢？

# 数据范围

  • 1≤n≤1001 \le n \le 1001≤n≤100

# 题解

我们观察有，满足条件的 a b 是 1 2 , 3 4 , 5 6 ... 连续的，而小 BBB 想要赢，只有 0 .... (1+2), (3+4), (5+6) 这样结尾的才行，因为这样末尾的 k+k+1k+k+1k+k+1 才能和第一个 000 凑成一对，而中间的全是 i+i+1i + i + 1i+i+1，小 AAA 拿一个，小 BBB 就拿另一个，这样小 BBB 就一定赢，其他情况都是输

# C

# 题目大意

给你一个由不同整数组成的数组 aaa，在一次操作中，你可以

  • 选择 aaa 的非空前缀，并用它的最小值替换
  • 选择 aaa 的非空后缀，并用它的最大值替换

对于每一个元素 aia_iai​，是否能通过若干次的上面的操作，使得数组变成 [ai][a_i][ai​] 只有它一个？

# 数据范围

  • 1≤n≤2×1051 \le n \le 2 \times 10^51≤n≤2×105

# 题解

维护一个 preMinpreMinpreMin 数组和 sufMaxsufMaxsufMax 数组，分别表示前 iii 个的最小值，和从 i→ni \to ni→n 的最大值
对于 aia_iai​ 来说，只要 aa[i] <= preMin[i - 1] || a[i] >= sufMax[i + 1] 即可完成题目要求

# D

# 题目大意

给小 AAA 和小 BBB 一个长度为 nnn 的二进制字符串 sss 和整数 k(1≤k<n)k~(1 \le k < n)k (1≤k<n)
如果小 AAA 能将 sss 中的字符全部转换为 000，那么他就赢了，反之小 BBB 赢

他们两个轮流操作，小 AAA 先

  • 如果是小 AAA 的轮次，那么在 sss 中选择长度为 kkk 的任意子序列（可以不连续），然后把这个子序列中的字符全部变成 000
  • 如果是小 BBB 的轮次，那么在 sss 中选择长度为 kkk 的任意子串（必须连续），然后把这个子串中的全部字符全部变成 111

只要在操作过程中，全部变成了 000，那么小 AAA 就赢了，请问当给定 sss 和 kkk 的时候，谁会赢？

# 数据范围

  • 1≤t≤1041 \le t \le 10^41≤t≤104
  • 2≤n≤2×1052 \le n \le 2 \times 10^52≤n≤2×105

# 题解

首先我们可以统计 111 的个数为 cntcntcnt

  • 如果 cnt≤kcnt \le kcnt≤k，那么小 AAA 一步就可以把字符串变为满足题目要求的字符串，故 AAA 能赢
  • 如果 cnt>kcnt > kcnt>k，我们发现，只要把原字符串的范围，能拆成 [k],[n−2k],[k][k], [n-2k], [k][k],[n−2k],[k]，只要中间的 n−2k≥0n-2k \ge 0n−2k≥0，那么小 BBB 一定赢
    • 为什么捏？因为当我们的 cnt>kcnt > kcnt>k 时，小 AAA 第一次操作，一定可以变为 cnt−kcnt - kcnt−k 个 111，那么这 cnt−kcnt - kcnt−k 个 111 肯定是分布在三个块之间的，小 BBB 此时的最优策略，肯定是找左右两边 111 较少的长度为 kkk 的段，把他们全部变为 111，那么此时场上肯定会出现，一段 kkk 全是 111，剩下两段里至少有 111 的情况，此时小 AAA 就不可能赢了
