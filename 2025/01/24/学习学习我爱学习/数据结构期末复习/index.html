# 绪论

  • 每种数据结构都具备三个基本运算：插入、删除、查找。 \color{red}
    • 比如二维数据就没有插入和删除
  • 逻辑结构不相同的数据，可以采用多种不同的存储方法。
  • 原地工作算法指的是需要的临时空间的大小不依赖问题规模 nnn，空间复杂度为 O(1)O(1)O(1)。
  • 数据结构在计算机内存中的表示是指数据的存储结构
  • 算法分析的主要任务之一是分析算法的执行时间与问题规模之间的关系。
  • 算法分析的目的是：分析算法的效率以求改进。
  • 算法的可行性是指指令不能有二义性。 \color{red}
    • 可行性是指：算法中的所有操作可以通过已经实现的基本操作之执行有限次来实现。
  • 数据的运算与采用何种存储结构有关。
  • 在规定的时间内完成不是算法的基本特性。
  • 时间复杂度为 O(x)O(x)O(x) 表示的是执行时间与 xxx 成正比。

# 链表

  • 对于一个具有 n 个元素的线性表，建立其单链表的时间复杂度是 O(n)O(n)O(n)
  • 在长度为 n 的带表头指针的、不带表头结点的循环单链表上，删除首元素的复杂度是 O(n)O(n)O(n)
  • 线性表的顺序存储结构和链式存储结构相比，优点是便于随机存取
  • 线性表的顺序存储结构和链式存储结构相比，优点是所有的操作算法实现简单 \color{red}
  • 线性表是一个有限序列，可以为空
  • 静态链表是使用顺序表（数组）作为存储形式的链表，结构体的指针域换成了记录下表
  • 要求采用静态空间分配方式，且插入和删除操作时不需要移动元素，采用的存储结构是静态链表
  • 如果对含有 n(n>1)n~(n>1)n (n>1) 个元素的线性表的运算只有 444 种，即删除第一个元素、删除尾元素、在第一个元素前面插入新元素、在尾元素的后面插入新元素，则最好使用只有开始数据节点指针没有尾节点指针的循环双链表

# 循环队列

循环队列是以顺序表来实现的

  • front 队首指针指向队首元素的前一位置， rear 队尾指针指向队尾元素
    • 队满条件： (rear + 1) % maxsize == front
    • 判空： front(头) == rear(尾)
  • 入队： rear = (rear + 1) % maxsize
  • 队满： front == (rear + 1) % maxsize
  • 计算元素个数：
    • rear > front ： 个数 = rear - front
    • rear < front ： 个数 = rear - front + maxsize
    • 综上， 元素个数 = (rear - front + maxsize) % maxsize

# 树的计算题

  • 一棵完全二叉树的第 666 层上有 232323 个叶子节点，则此二叉树最多有多少个结点？
    • 完全二叉树的叶子结点只能在最下两层，要使结点最多，这棵二叉树深度为 777，前 666 层结点数共为 636363，第 666 层有 323232 个结点，其中叶子为 232323 个，非叶子为 999 个，它们的度都为 222，第 777 层只有 181818 个结点，故整棵二叉树结点数为 818181
    • 如果是求 最少的结点数，则让这 232323 个叶子结点靠左（即第六层就是最后一层，求最多时是让这 232323 个结点靠右，第六层是倒数第二层）
  • 设深度为 hhh 的二叉树中只有度为 000 和度为 222 的结点，则此类二叉树中所包含结点数至少为？
    • 除根之外，每层只有两个结点，且互为兄弟，有 2*(h - 1) + 1 = 2h - 1 个点
    • 若求的是至多，则即为满二叉树，有 2h−12^{h} - 12h−1 个点
  • 一棵深度为 kkk 且只有 kkk 个结点的二叉树按照完全二叉树顺序存储的方式存放于一个一维数组 R[n]R[n]R[n] 中，则 nnn 至少是？
    • 二叉树最恶劣情况是，全部右斜放，则 R[n]R[n]R[n] 相当于应该有满 kkk 层的个数，即 2k−12^k-12k−1
    • 最好情况是，二叉树全部左斜放，用的点数最少，即为 2k−1−1+1=2k−12^{k-1}-1+1=2^{k-1}2k−1−1+1=2k−1，但题目问的是至少，为了防止不越界，应取 2k−12^k-12k−1
  • 用二叉链表表示具有 nnn 个结点的二叉树时，值为空的指针域的个数为 n+1n+1n+1
  • 二叉树的先序遍历序列和后序遍历序列正好相反，则该二叉树一定满足的条件是：高度等于结点数
  • 若二叉树采用二叉链表存储结构，要交换所有分支结点的左右子树的位置，利用基于后序遍历的递归算法最合适
  • 基于中序线索化链表，其头结点指针为 headheadhead，对应的二叉树为空的判断条件是： head->left = head && head->right == head
  • 二叉树的先序遍历的递归算法的时间复杂度为线性级

# 二叉排序树的删除

假设被删结点为 ∗p*p∗p，其父亲结点是 ∗f*f∗f，PL,PRP_L,~P_RPL​, PR​ 分别代表其左子树和右子树

  • 如果 ∗p*p∗p 是叶子节点，直接删除该点即可， f->lChild = NULL 或 f->rChild = NULL
  • 如果 ∗p*p∗p 只左子树 PLP_LPL​ 或右子树 PRP_RPR​，直接删除该点即可， f->lChild = p->lChild 或 f->rChild = p->rChild
  • 如果 ∗p*p∗p 左右子树均不空，则有两种处理方法
    • 中序遍历二叉树得到的序列为 \
    1. 令 ∗p*p∗p 左子树为 ∗f*f∗f 左子树，而 ∗p*p∗p 右子树为 ∗s*s∗s 的右子树， f->lChild = p->lChild, s->rChild = p->rChild ，sss 是，在中序遍历，∗p*p∗p 的前一个结点（它一定是一个叶子结点）
    2. 令 ∗p*p∗p 的直接前驱（或直接后继）代替 ∗p*p∗p，然后再从二叉排序树中删除它的直接前驱（或直接后继）（递归删除，可能要删多次）
    • 显然，前一种处理方法可能增加树的深度，而后一种方法是以被删结点左子树中关键字最大的结点替代被删结点，然后从左子树中删除这个结点。此结点一定没有右子树，这样就不会增加树的深度

# 线性再散列

线性再散列的基本思想是将散列表划分为若干个子表，当一个关键字在某个子表中发生冲突时，通过线性探测 (即向后移动固定的步长) 来寻找下一个空位置

假设每个子表中最多有 kkk 个关键字，那么查找其中任意一个同义词关键字的比较次数将是这些子表中关键字的比较次数之和
