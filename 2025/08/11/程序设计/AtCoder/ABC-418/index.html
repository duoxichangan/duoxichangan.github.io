# B

# 题目大意

给定一个字符串 SSS，ttt 是其子串，如果 t0t_0t0​ 和 tlastt_{last}tlast​ 都是字符 t 的话，它就会有一个值，字符t的个数−2字符串长度−2\frac{字符t的个数 - 2}{字符串长度 - 2}字符串长度−2字符t的个数−2​
请你求出最大的这个值

# 题解

  • 1≤len≤1001 \le len \le 1001≤len≤100

# 题解

其实题目很容易，但是为什么写在这呢？
是因为，在写这题的时候，用了 vectorvectorvector，在其大小为 000 的时候，我访问了 idx.size() - 1
此时会有一个神奇的事情发生，这个的结果并不是 −1-1−1，而是一个超级大的数，为什么呢？
这是由于 .size().size().size() 的类型是无符号整数，当 −1-1−1 后，直接溢出了，进而会造成 TLETLETLE

# C

# 题目大意

你有 nnn 个茶包，第 iii 个茶包里有 AiA_iAi​ 包茶，口味为 iii
现在你和庄家做一个游戏，游戏难度在 [1,∑i=1nAi][1, \sum\limits_{i=1}^{n} A_i][1,i=1∑n​Ai​]，假设为 bbb

  • 你申明一个整数 xxx，b≤x≤∑i=1nAib \le x \le \sum\limits_{i=1}^{n} A_ib≤x≤i=1∑n​Ai​
  • 庄家从桌子上选 xxx 包茶给你
  • 你从 xxx 包茶中选择 bbb 包
  • 如果这 bbb 包都是一个口味，那么你赢了

你现在有 QQQ 个问题，第 jjj 个问题是

  • 一个难度为 BjB_jBj​ 的游戏，你能获胜的最小整数 xxx 是多少（必须赢）

# 题解

我要从你给我的一堆茶中，选 bbb 包同样口味的
站在庄家的角度
→\to→ 如果一开始，某个茶包中的茶数量 <b< b<b，庄家可以把这个茶包里的茶全部给我；然后可以找到第一个 ≥b\ge b≥b 的茶包（ idxidxidx ），1→idx−11 \to idx - 11→idx−1 都可以给我 →\to→ 二分找 bbb、前缀和计算前 idx−1idx - 1idx−1 个
接下来，是不是有 n−bn - bn−b 个茶包里面的茶 ≥b\ge b≥b，如果给我 (n−b)∗(b−1)(n-b) * (b-1)(n−b)∗(b−1) 包，那我就赢不了
也就是说，再多给我一包，我就赢了
xxx = pre[idx - 1] + (n - b) * (b - 1) + 1

# 题目大意

给你一个长度为 nnn 的，只由 0/10/10/1 组成的字符串 sss，你可以执行任意次一下操作

  • 任选两个相邻的字符，两者相同则变为 111，否则变为 000

如果一个字符串能通过若干次这种操作，变为一个 111，则称这个字符串为好字符串
现在给你一个字符串，问其中，有多少个字串是好字符串

# 数据范围

  • 1≤n≤1051 \le n \le 10^51≤n≤105

# 题解

不难发现，只要字符串中，000 的个数是偶数个，那么这个字符串就是好字符串
利用 桶 的思想，计算个数
