# D

# 题目大意

给你一个由 000 和 111 组成的字符串，你可以进行一种操作：交换相邻字符
问，若想要让所有 111 都相邻，最少的操作次数是多少

# 解题思路

贪心贪心贪心，这题本质就是货仓选址，但是与货仓选址不同的是，需要考虑到，每个需要移动的 111 并不是移动到中间的 111 的位置，而挪到距离中间的 111 可能有一些距离

# E

# 问题大意：

我们有一个数组 AAA 和一个整数 KKK，要求对于数组中的每个元素 AiA_iAi​，找到它的最大公约数 GCDGCDGCD，且满足以下两个条件：

  • AiA_iAi​ 可以被 GCDGCDGCD 整除
  • 在数组中至少有 KKK 个元素能够被这个 GCDGCDGCD 整除
    我们需要找出最大的 GCDGCDGCD，即对于每个 AiA_iAi​，找出一个能使 AiA_iAi​ 与 KKK 个元素的 GCDGCDGCD 最大的值

# 解题思路

# 统计每个数的出现次数

首先，定义一个数组 s[] 来记录每个数在数组 AAA 中的出现次数。假设最大元素为 MMM，即 M = max(A)
通过一次遍历 AAA，我们可以完成这个统计操作，时间复杂度为 O(N)O(N)O(N)，其中 NNN 是数组 AAA 的长度

# 计算每个数的倍数出现次数

接下来，我们要计算每个数在数组中所有倍数的元素个数。我们定义一个数组 t[] ，其中 t[d] 表示所有能够被 ddd 整除的元素的个数，即：

td=∑d∣nsnt_d = \sum_{d \mid n} s_ntd​=d∣n∑​sn​

具体的做法是，对于每个数从 111 到 MMM，遍历它的倍数 (d,2d,3d,…)( d, 2d, 3d, \dots )(d,2d,3d,…)，将 s[j] 加到 t[d] ，其中 jjj 是 ddd 的倍数

# 时间复杂度

这个过程的时间复杂度是 O(Mlog⁡M)O(M \log M)O(MlogM)，因为我们遍历每个数的倍数，而倍数的个数是按对数增长的
